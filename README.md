# E-Commerce Website Selenium Testing Framework

A Java-based end-to-end test automation framework for an e-commerce website, built with Selenium WebDriver, TestNG, and ExtentReports. This project follows the Page Object Model (POM) design pattern and supports data-driven testing via Apache POI.

## Configuration

The `testRunner.xml` file at the project root controls which test classes and suites are executed:

```xml
<suite name="Suite">
  <listeners>
    <listener class-name="reporter.ExtentReportsListener" />
  </listeners>
  <test thread-count="5" name="Test">
    <classes>
      <class name="testCases.TestCases"/>
    </classes>
  </test>
</suite>
```

- To add new test classes, add additional `<class>` entries inside the `<test>` block.
- The `thread-count` attribute controls parallel execution (currently set to 5).
- The `ExtentReportsListener` automatically generates an HTML report after each run.

## Running Tests

### Run all tests via Maven

```bash
mvn test
```

### Run using the TestNG XML suite

```bash
mvn test -DsuiteXmlFile=testRunner.xml
```

### Run a specific test class

```bash
mvn test -Dtest=TestCases
```

## Test Reports

After each test execution, an HTML report is automatically generated by **ExtentReports** via the `ExtentReportsListener` TestNG listener.

- Reports are generated in the `test-output/` directory (or wherever configured in the listener).
- Open the generated `.html` file in any browser to view a detailed, interactive test report including pass/fail status, screenshots (if configured), and execution timeline.

## Framework Design

This framework is built on the following principles:

**Page Object Model (POM)**
UI interactions are encapsulated in dedicated Page classes. Tests interact only with page methods, keeping test logic clean and maintainable.

**Data-Driven Testing**
Apache POI is used to read test data from Excel (`.xlsx`) files, enabling the same test to run with multiple data sets without code changes.

**Automatic Driver Management**
WebDriverManager eliminates the need to manually download or configure browser drivers. The correct driver version is automatically resolved at runtime.

**Reporting**
ExtentReports generates rich HTML reports with step-by-step details, making it easy to review test results locally or share with stakeholders.

**TestNG Integration**
TestNG handles test grouping, parallel execution, and lifecycle annotations (`@BeforeClass`, `@AfterMethod`, etc.) for clean setup and teardown.

## Contributing

This project is actively evolving. New test cases are being added regularly.

To add new tests:

1. Create a new Page Object class under `src/main/java/` for the target page.
2. Add test methods to `testCases/TestCases.java`, or create a new test class.
3. Register the new class in `testRunner.xml` under the `<classes>` block.
4. Follow existing naming conventions and use the base class / utility methods where applicable.
